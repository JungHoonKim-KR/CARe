## 1. Privy (소셜 인증 / 임베디드 지갑)

Privy는 렌터 앱의 소셜 로그인(Google, Apple 등)과 임베디드 지갑 생성에 사용됩니다.

### 1.1 계정 생성 및 앱 등록

1. [https://privy.io](https://privy.io) 접속 → **Get started** 클릭
2. 이메일로 회원가입 후 로그인
3. 대시보드 → **Create new app** 클릭
4. App Name: `CARe` (임의 입력)
5. 앱 생성 후 **Settings** 탭 이동

### 1.2 APP_ID / APP_SECRET 발급

**Settings → Basics** 탭에서 확인:

| 항목 | 설명 | 환경 변수 |
|------|------|----------|
| App ID | `app_xxxxxxxxxxxx` 형태 | `PRIVY_APP_ID` |
| App Secret | **Server Secret** 섹션에서 확인 | `PRIVY_APP_SECRET` |

> **App Secret**은 생성 후 다시 볼 수 없으므로 즉시 복사해 안전한 곳에 보관하세요.

### 1.3 로그인 방법 설정

**Settings → Login methods** 탭:

- **Recommended 활성화**: Email, Google, Apple
- **Wallet**: Embedded Wallet 활성화
  - Wallet type: `Ethereum`
  - Create on login: `Enabled` (자동 지갑 생성)

### 1.4 허용 도메인 설정

**Settings → Allowed origins** 탭에서 다음 도메인 추가:

```
https://j14e201.p.ssafy.io
http://localhost:5174
```

### 1.5 환경 변수 반영

`backend/.env`:
```bash
PRIVY_APP_ID=app_xxxxxxxxxxxx
PRIVY_APP_SECRET=your_app_secret
PRIVY_SERVER_URL=http://privy-server:3001   # 프로덕션
# PRIVY_SERVER_URL=http://localhost:3001    # 로컬
```

`renter/.env` (privy-server.mjs 전용):
```bash
PRIVY_APP_ID=app_xxxxxxxxxxxx
PRIVY_APP_SECRET=your_app_secret
```

`renter/.env.local`:
```bash
VITE_PRIVY_SERVER_URL=http://localhost:3001
```

## 2. Pinata (IPFS 클라우드)

흠집 이미지의 무결성 증명을 위한 IPFS 파일 고정(pinning) 서비스입니다.

### 2.1 계정 생성

1. [https://pinata.cloud](https://pinata.cloud) 접속 → **Sign Up**
2. 이메일로 회원가입 후 이메일 인증

### 2.2 API 키 발급

1. 대시보드 우측 상단 → **API Keys** 탭
2. **New Key** 클릭
3. 설정:
   - Key Name: `care-backend`
   - Admin: `ON` (전체 권한) 또는 필요한 권한만 선택
     - `pinFileToIPFS` ✅
     - `pinJSONToIPFS` ✅
4. **Generate API Key** 클릭
5. 팝업에서 세 가지 값 즉시 복사 (이후 재확인 불가):

| 항목 | 환경 변수 |
|------|----------|
| API Key | `PINATA_API_KEY` |
| API Secret | `PINATA_API_SECRET` |
| JWT | `PINATA_JWT` |

### 2.3 환경 변수 반영

`backend/.env` 및 `ai/verify/.env`, `ai/yolo/.env`:
```bash
PINATA_API_KEY=your_pinata_api_key
PINATA_API_SECRET=your_pinata_api_secret
PINATA_JWT=your_pinata_jwt
```

> 프로젝트 내 AI 서버는 IPFS 업로드 시 `PINATA_JWT`만 사용합니다.

### 2.4 무료 플랜 제한

| 항목 | 무료 플랜 |
|------|---------|
| 저장 용량 | 1 GB |
| 업로드 | 월 100건 |
| Gateway | 공용 게이트웨이 (`gateway.pinata.cloud`) |

테스트 용도로는 무료 플랜으로 충분합니다.

## 3. AWS S3 + CloudFront (이미지 스토리지)

스캔된 차량 흠집 이미지(원본 + 크롭)를 S3에 저장하고 CloudFront로 배포합니다.

### 3.1 AWS 계정 생성

1. [https://aws.amazon.com/ko](https://aws.amazon.com/ko) → **계정 생성**
2. 신용카드 등록 후 이메일 인증 (프리 티어 12개월 무료)

### 3.2 S3 버킷 생성

1. AWS 콘솔 → **S3** → **버킷 만들기**
2. 설정:
   - 버킷 이름: `care-scratch-images` (고유한 이름)
   - 리전: `ap-northeast-2` (서울)
   - **모든 퍼블릭 액세스 차단**: ✅ (CloudFront를 통해서만 접근)
3. **버킷 만들기** 클릭

#### CORS 설정

버킷 → **권한** 탭 → **CORS(Cross-origin 리소스 공유)** → 편집:

```json
[
  {
    "AllowedHeaders": ["*"],
    "AllowedMethods": ["GET", "PUT", "POST"],
    "AllowedOrigins": [
      "https://j14e201.p.ssafy.io",
      "http://localhost:5174",
      "http://localhost:3000"
    ],
    "ExposeHeaders": ["ETag"]
  }
]
```

### 3.3 IAM 사용자 생성 및 권한 부여

1. AWS 콘솔 → **IAM** → **사용자** → **사용자 생성**
2. 사용자 이름: `care-s3-user`
3. **권한 설정** → **정책 직접 연결** → **AmazonS3FullAccess** 선택
   (또는 아래 최소 권한 정책을 직접 작성)

#### 최소 권한 인라인 정책 (권장)

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:PutObject",
        "s3:GetObject",
        "s3:DeleteObject",
        "s3:ListBucket"
      ],
      "Resource": [
        "arn:aws:s3:::care-scratch-images",
        "arn:aws:s3:::care-scratch-images/*"
      ]
    }
  ]
}
```

4. 사용자 생성 후 → **보안 자격 증명** 탭 → **액세스 키 만들기**
   - 사용 사례: `애플리케이션 실행 코드`
5. **액세스 키 ID**와 **비밀 액세스 키** 즉시 복사 (이후 재확인 불가)

### 3.4 CloudFront 배포 생성

1. AWS 콘솔 → **CloudFront** → **배포 생성**
2. 설정:
   - **원본 도메인**: 생성한 S3 버킷 선택
   - **S3 버킷 액세스**: `원본 액세스 제어 설정(OAC)` 권장
     - 새 OAC 생성 → 이름: `care-oac`
   - **기본 캐시 동작**:
     - Viewer Protocol Policy: `Redirect HTTP to HTTPS`
     - Cache Policy: `CachingOptimized`
3. **배포 생성** 후 배포 도메인 이름 복사 (예: `xxxx.cloudfront.net`)
4. S3 버킷 정책 업데이트 (OAC 사용 시 CloudFront가 자동 안내)

### 3.5 환경 변수 반영

`backend/.env`, `ai/verify/.env`, `ai/yolo/.env`:
```bash
AWS_ACCESS_KEY_ID=AKIA...
AWS_SECRET_ACCESS_KEY=your_secret_key
AWS_REGION=ap-northeast-2
S3_BUCKET_NAME=care-scratch-images
CLOUDFRONT_DOMAIN=xxxx.cloudfront.net
```

> CloudFront 도메인은 `https://` 없이 도메인만 입력합니다.

## 4. 스마트 컨트랙트 배포 (Hardhat)

프로젝트에는 4개의 스마트 컨트랙트가 있으며 SSAFY 블록체인 네트워크에 배포합니다.

| 컨트랙트 | 설명 | 환경 변수 |
|----------|------|----------|
| `CareToken` | CARE ERC-20 토큰 (보증금/정산 통화) | `MOCK_USDC_ADDRESS` |
| `CarNFT` | 차량 NFT (차량 등록 증명) | `CAR_NFT_ADDRESS` |
| `DIDRegistry` | DID 온체인 등록/관리 | `DID_REGISTRY_ADDRESS` |
| `DisputeSettlement` | 분쟁 정산 합의 기록 | `DISPUTE_SETTLEMENT_ADDRESS` |

> **RentalContract**는 현재 별도 배포 스크립트 없음 — `RENTAL_CONTRACT_ADDRESS`는 사용 시 수동 배포 후 입력.

### 4.1 사전 준비

#### Node.js 설치

```bash
# Node.js 20.x LTS 설치 (nvm 사용 권장)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
nvm install 20
nvm use 20
```

#### 의존성 설치

```bash
cd blockchain
npm install
```

#### 배포 지갑 준비

SSAFY 블록체인에 배포하려면 배포자 지갑의 **Private Key**와 해당 네트워크 **가스비용 잔액**이 필요합니다.

- MetaMask에서 새 계정 생성 → Private Key 내보내기
- SSAFY 네트워크 Faucet에서 테스트용 ETH 수령
  - SSAFY 블록체인 RPC: `https://rpc.ssafy-blockchain.com`
  - Chain ID: `31221`

`backend/.env`에 Private Key 설정 (hardhat.config.js가 `../backend/.env` 공유):
```bash
PRIVATE_KEY=0xabcdef...   # 0x 포함 64자리 hex
RPC_URL=https://rpc.ssafy-blockchain.com
CHAIN_ID=31221
```

### 4.2 컨트랙트 컴파일

```bash
cd blockchain
npx hardhat compile
```

컴파일 성공 시 `blockchain/artifacts/` 디렉토리에 ABI 파일이 생성됩니다.

### 4.3 컨트랙트 배포

#### 방법 A: Hardhat Ignition 사용 (권장)

```bash
cd blockchain

# 1. CareToken (CARE 토큰) 배포
npx hardhat ignition deploy ignition/modules/CareToken.js --network ssafy

# 2. CarNFT 배포
npx hardhat run scripts/deploy_car_nft.js --network ssafy

# 3. DIDRegistry 배포
npx hardhat ignition deploy ignition/modules/DIDRegistry.js --network ssafy

# 4. DisputeSettlement 배포
npx hardhat ignition deploy ignition/modules/DisputeSettlement.js --network ssafy
```

#### 방법 B: 배포 스크립트 직접 실행

```bash
# CareToken
npx hardhat run scripts/deploy_care.js --network ssafy

# CarNFT
npx hardhat run scripts/deploy_car_nft.js --network ssafy
```

### 4.4 배포 후 주소 확인 및 환경 변수 반영

배포 완료 후 콘솔에 출력되는 컨트랙트 주소를 `backend/.env`에 반영합니다:

```bash
CAR_NFT_ADDRESS=0x...
DID_REGISTRY_ADDRESS=0x...
MOCK_USDC_ADDRESS=0x...        # CareToken 주소
DISPUTE_SETTLEMENT_ADDRESS=0x...
RENTAL_CONTRACT_ADDRESS=0x...  # 별도 배포 시
```

Ignition 사용 시 배포 결과는 `blockchain/ignition/deployments/` 디렉토리에도 저장됩니다.

### 4.5 배포 후 CARE 토큰 Faucet (초기 잔액 지급)

테스트용 CARE 토큰을 지갑에 지급합니다.

`blockchain/scripts/faucet.js`의 환경 변수 설정:

```bash
# blockchain/.env 또는 backend/.env에 추가
MOCK_USDC_ADDRESS=0x...           # CareToken 컨트랙트 주소
RENTER_WALLET_ADDRESS=0x...       # 렌터 지갑 주소
COMPANY_WALLET_ADDRESS=0x...      # 업체 지갑 주소
BLOCKCHAIN_WALLET_ADDRESS=0x...   # 서버 운영 지갑 주소
```

```bash
npx hardhat run scripts/faucet.js --network ssafy
```

기본 지급량:
- 렌터: `150,000 CARE`
- 업체: `10,000 CARE`
- 서버: `1,000 CARE`

### 4.6 잔액 확인

```bash
npx hardhat run scripts/check_balance.js --network ssafy
```

## 5. APICK API (차량 번호판 조회)

차량 번호판으로 차량 정보를 조회하는 공공 API 서비스입니다.

### 5.1 가입 및 API 키 발급

1. [https://apick.app](https://apick.app) 접속 → 회원가입
2. 로그인 후 **API 목록** → **자동차 정보 조회** 검색
3. 해당 API 구독 신청
4. **마이페이지** → **API 키 관리** 에서 발급된 키 확인

### 5.2 환경 변수 반영

`backend/.env`:
```bash
APICK_API_KEY=your_apick_api_key
```

## 6. NTS API (국세청 사업자등록번호 조회)

업체의 사업자등록번호 진위 확인에 사용합니다.

### 6.1 가입 및 API 키 발급

1. [https://www.data.go.kr](https://www.data.go.kr) (공공데이터포털) 접속 → 회원가입
2. 검색창에 `국세청_사업자등록정보 진위확인` 검색
3. 해당 API → **활용 신청** 클릭
4. 신청 승인 후 **마이페이지** → **개발계정** → API 키(일반 인증키) 확인

> 승인까지 보통 1~2 영업일 소요됩니다.

### 6.2 환경 변수 반영

`backend/.env`:
```bash
NTS_API_KEY=your_nts_api_key
```

## 7. GMS (문자 발송 서비스)

예약 확인, 알림 등 SMS 발송에 사용합니다.

### 7.1 가입 및 API 키 발급

1. [https://gms.show](https://gms.show) 접속 → 회원가입
2. 로그인 후 **API 관리** → **API 키 발급**
3. 발급된 API Key 복사

### 7.2 발신번호 등록

SMS 발송을 위해 발신번호를 사전 등록해야 합니다:

1. **발신번호 관리** → **발신번호 등록**
2. 인증 절차(통신사 확인) 완료 후 등록

### 7.3 환경 변수 반영

`backend/.env`:
```bash
GMS_KEY=your_gms_api_key
```
