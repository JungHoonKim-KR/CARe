## 💡 새롭게 배운 점

### Process
- 프로그램의 하나의 인스턴스가 프로세스 (프로그램이 실제 메모리에 코드가 실행된 상태)
 
- 프로그램은 하나지만, 같은 프로세스가 여러 개가 될 수 있다. 

- 프로세스는 반드시 하나 이상의 스레드를 가짐

### Thread
- 프로세스 내에서 실행되는 흐름의 단위
- CPU 는 한번에 하나의 명령 수행 

### Multi Thread의 필요성
-10개의 요청이 동시간대에 이루어진다면? 

#### 4. Sleep을 이용한 쓰레드 생성
- Thread를 실행할 때 일정 텀을 되서 데이터가 공유되어도 문제가 없음

- 단, Sleep 되어진 시간 보다 쓰레드에서 데이터를 처리하는 시간이 길어질 경우 데이터가 꼬일 수 있음 

### Multi Thread 동작 원리
- 하나의 쓰레드가 공유 자원을 접근 할 경우 반드시 lock (mutex, semaphore 등..) 을 걸어야 한다.

- 특수한 상황 (알 수 없는 버그등)으로 인해 unlock이 되지 않으면 그 무서운 Dead Lock 발생
