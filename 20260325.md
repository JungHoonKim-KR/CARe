## 6. 중복 코드 / 구조 문제

### 6-1. `scanAfter` 함수 중복 정의
- `reservation.js` (line 83~92): base64 dataUrl → blob 변환 후 multipart 전송
- `scan.js` (line 22~28): imageBlob 직접 받아서 multipart 전송
- `scanner.js`에서는 `scan.js`의 함수를 import
- `CarReturnPage`에서는 `reservation.js`의 함수를 import
- **문제**: 같은 API를 두 곳에서 다르게 정의. 하나로 통일 필요

### 6-2. CarCrackPage의 4방향 vs ScanPage의 6방향
- `zones.js`: 6방향 정의 (rear-right, front-right, front, front-left, rear-left, rear)
- `CarCrackPage.jsx`: 4방향만 정의 (front, rear, left, right)
- `CarReturnPage.jsx`: 4방향만 정의 (front, rear, left, right)
- **문제**: ScanPage에서 6방향으로 스캔한 결과가 CarCrackPage에서 4방향으로만 표시됨. 불일치.

### 6-3. `getReservationDetail` 미활용
- 함수가 정의되어 있지만 모든 페이지가 `location.state`로 reservation 데이터를 전달받음
- 새로고침 시 state가 없어져서 빈 화면 발생 가능
- **해결 방향**: 주요 페이지에서 state가 없을 때 fallback으로 API 호출
