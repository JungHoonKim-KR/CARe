## 블록 체인의 이해
혜린님이 정리해준 내용이 이해하기 쉬워서 인용해왔다.

### 체인 / 코인 / 토큰의 개념
1. 체인 : 하나의 시스템, 혹은 운영체제 (ex. 이더리움, 비트코인, 폴리곤, 솔라나 등)

2. 코인 : 우리가 흔히 아는 암호 화폐. 각 체인마다 고유한 코인을 갖고 있음. 즉 체인이 없으면 코인이 존재할 수 없다 (ex. 이더리움 -> ETH / 비트코인 -> BTC / 솔라나 -> SOL)

3. 토큰 : 체인 위에서 스마트 컨트랙트로 만들어진 자산. 체인 운영자나 개발자 등이 만드는 게 아니고, 해당 체인을 활용해서 모두가 생성 가능. NFT가 여기에 해당됨.

++ 참고로 우리가 체인을 개발하지 않는 이상, 저희가 생성하는 NFT나 스테이블 코인은 전부 토큰의 특성을 가진다.

```
(예시)
체인 : 동물의 숲
코인 : 동물의 숲에서 사용되는 고유 화폐인 >> 벨 <<
토큰 : 동숲 내 아이템 정도(비록 동숲 유저는 아이템을 만들 수 없지만,,,,) 거래는 할 수 있음
```


### AI 강의 LLM 작동방식에서의 포인트

```
┌───────┐   ┌──────────┐   ┌──────────────────┐   
│ Input │ → │ Tokenize │ → │ Token Embedding  │ → 
└───────┘   └──────────┘   └──────────────────┘   
┌──────────┐   ┌─────┐   ┌───────────────┐   ┌────────────────────┐
│ Encoding │ → │ LLM │ → │ Output (Logit)│ → │ Sampling & Decoding│
└──────────┘   └─────┘   └───────────────┘   └────────────────────┘
```


- LLM의 output은 `문장`이 아니라 `logit`이다. 즉, `Raw Probability`가 나온다. 매 순간마다 분류를 하고 있음.

## 블록체인 세션 복습

### 개념

| 데이터를 분산 저장 + 처리하는 기술. 데이터를 블록에 엮인 형태로 이어짐. 시간의 흐름이 존재, 순차적으로 연결.

    ex. 마법 일기장. 누군가가 쓰면 내 일기장에도 내용 그대로 써짐.

### 특징
1. 탈중앙성: 얼마나 자율적인가?

    ```
    ex. 은행은 중앙집중식. 블체는 우리가 정하는 대로!
    ```
    - 내결함성: (이더리움 기준) 내부에서 생기는 결함에 강하다. 51%룰. 
    - 공격/담합 저항성: 내결함성에서 이어지는 내용. 다 같이 똑같은 걸 갖고 있으니, 하나만 공격해서는 바꿀 수 없어서 공격/담합이 어렵다.

2. 확장성: 얼마나 빨리 + 많이 처리되는가. 확장성과 보안성이 높으면 탈중앙화의 성격이 비교적 약하다는 트릴레마와 연결(국가/기업 주도)
3. 보안성 = 안전성

4. 블록체인 트릴레마
탈중앙성(오래 걸림), 보안성(보안성은 있지만 확장이 어려움 시간 개걸림), 확장성(빠르지만 시간 오래 걸림) 이 모두 만족하기 어렵다!

### 용어

1. 노드: 네트워크에 참여해서 장부를 기록하는 개별 컴퓨터.
    ```
    ex. 마법 학교 일기장을 작성하는 개개인.
    ```
2. 코인: 현실 화폐로 교환할 수 있음.
    ```
    ex. 학교 식권. 체인에서 생성하는 화폐라 개인이 발급할 수 없다.
    ```
3. FT(Fundable Token): 여러 개 존재 가능하며, 내부룰에 따라 결정되는 개념. 참고로 `스테이블 코인`은 코인이 아니라 `토큰`이다.

4. NFT(Non-Fundable Token): 고유한 번호와 데이터가 존재하는 토큰.

    ```
    ex. 수제로 만들어진, 세상에 단 하나 뿐인 작품 등
    ```

5. 합의: 여러 노드가 하나의 동일한 블록체인 상태(어떤 거래가 유효하고, 어떤 블록이 진짜인지)에 동의하도록 만드는 메커니즘. 중앙 서버가 없으니, 네트워크 참여자들끼리의 룰을 정하는 알고리즘/규칙의 집합.

6. 스마트 컨트랙트: 사전에 협의한 내용을 미리 프로그래밍해서, 조건이 충족되면 자동으로 계약이 이행.

7. DAO: 탈중앙자율조직. 모두가 제안하고 투표 가능하며, 그 결과에 따라 스마트 컨트랙트가 자동으로 실행됨.
