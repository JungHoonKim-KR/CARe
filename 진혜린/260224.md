# JIRA 사용법 정리 및 운영 컨벤션 확립

---

## 1. 이슈 계층 구조

### 📌 구조

```
Level 1   에픽 (Epic)
Level 2     └── 스토리 (Story) / 버그 (Bug)
Level 3           └── 작업 (Task)
```

### 📌 규칙

- 에픽은 큰 기능 단위를 묶는 컨테이너
- 스토리는 에픽 하위에서 **사용자 관점 서술형**으로 작성
- 버그는 에픽 하위에 속함 (드물게 사용)
- 작업은 스토리/버그 하위에서 **개발 관점**으로 작성
- 작업은 **절대 하위 작업(Subtask)으로 생성 금지**
  - 반드시 **작업(Task)** 으로 생성 후 상위 스토리/버그에 연동
- 에픽 없이 스토리/버그/작업 단독 생성 금지
- 스토리/버그 없이 작업 단독 생성 금지

## 2. 레이블 정의 (파트 구분)

| 레이블 | 의미 |
|--------|------|
| FE | 프론트엔드 |
| BE | 백엔드 |
| INF | 인프라 |
| BC | 블록체인 |
| ETC | 기타 (기획, 설계, 회의 등) |

### 📌 규칙

- 레이블은 **작업(Task)에만 설정**
- 에픽 / 스토리 / 버그에는 레이블 설정 ❌

## 3. 네이밍 컨벤션

---

### 3-1. Level 1 — 에픽 네이밍

**형식**

```
{번호}: {설명}
```

**규칙**

- 번호는 두 자리 (01, 02, 03 ...)
- 설명은 기능 단위로 간결하게 (한국어)
- 담당자 설정 ❌
- 레이블 설정 ❌
- Story Points 설정 ❌

**예시**

```
01: 회원 기능
02: 친구 기능
00: 기타 (기획/설계)
```

---

### 3-2. Level 2 — 스토리 네이밍

**형식**

```
{에픽번호}-S{세부번호}: {OO는 OO를 할 수 있다}
```

**규칙**

- 에픽번호는 상위 에픽 번호와 동일
- 세부번호는 두 자리 (01, 02, 03 ...)
- 반드시 사용자 관점 서술형
  - "OO는 OO를 OO할 수 있다"
  - 주체는 내부 회원 체계 기준 (유저 / 관리자 등)
- 담당자 설정 ❌
- 레이블 설정 ❌
- **Story Points 반드시 설정 (1 / 2 / 3 / 5 / 8)**
- **우선순위 반드시 설정 (Highest / High / Medium / Low / Lowest)**

**예시**

```
01-S01: 관리자는 로그인을 할 수 있다
01-S02: 유저는 이메일과 비밀번호로 회원가입을 할 수 있다
01-S03: 유저는 내 프로필 정보를 조회하고 수정할 수 있다
02-S01: 유저는 친구 목록을 조회할 수 있다
```

---

### 3-3. Level 2 — 버그 네이밍

**형식**

```
{에픽번호}-B{세부번호}: {발생한 문제 서술}
```

**규칙**

- 어떤 상황에서 어떤 문제가 발생했는지 명확하게 작성
- 우선순위 반드시 설정
- Story Points 설정 ❌

**예시**

```
01-B01: 로그인 토큰 만료 시 갱신 없이 오류가 반환되는 문제
03-B01: 회원 목록 페이징 처리 시 마지막 페이지 데이터 누락 문제
```

---

### 3-4. Level 3 — 작업 네이밍

**형식**

```
{에픽번호}-{스토리번호}-{세부번호}: {설명}
```

> 상위가 스토리 01-S01이면 → `01-S01-01`  
> 상위가 버그 01-B01이면 → `01-B01-01`

**규칙**

- 상위 스토리/버그 번호와 반드시 동일하게 맞춤
- 세부번호는 두 자리 (01, 02, 03 ...)
- 개발 관점으로 구체적으로 작성
- API의 경우 엔드포인트 명시 권장
- 담당자 반드시 설정
- 레이블 반드시 설정 (FE / BE / INF / BC / ETC)
- Story Points 설정 ❌
- 우선순위 반드시 설정
- 이슈 유형은 반드시 **작업(Task)**  
  → Subtask 절대 금지

**예시 (상위: 01-S01)**

```
01-S01-01: 관리자 로그인 API 구현 (POST /api/auth/login)   [BE] / 진혜린
01-S01-02: JWT 토큰 발급 및 갱신 처리                        [BE] / 정현우
01-S01-03: 관리자 화면 구성                                  [FE] / 강희정
01-S01-04: 로그인 폼 유효성 검사 구현                        [FE] / 김정훈
```

**예시 (상위: 01-B01)**

```
01-B01-01: 토큰 만료 예외 처리 및 자동 갱신 로직 수정   [BE] / 정현우
```

## 4. 전체 구조 예시 — 회원 기능

```
(에픽) 01: 회원
  │
  ├── (스토리) 01-S01: 관리자는 로그인을 할 수 있다
  │       SP: 3 / High
  │       ├── 01-S01-01 관리자 로그인 API 구현           [BE] / 진혜린
  │       ├── 01-S01-02 JWT 토큰 발급 및 갱신 처리        [BE] / 진혜린
  │       ├── 01-S01-03 관리자 화면 구성                  [FE] / 강희정
  │       └── 01-S01-04 로그인 폼 유효성 검사 구현        [FE] / 강희정
  │
  ├── (스토리) 01-S02: 유저는 이메일과 비밀번호로 회원가입을 할 수 있다
  │       SP: 5 / High
  │       ├── 01-S02-01 회원가입 API 구현                 [BE] / 손효지
  │       ├── 01-S02-02 이메일 중복 검사 API 구현         [BE] / 손효지
  │       ├── 01-S02-03 회원가입 화면 구현                [FE] / 조하원
  │       └── 01-S02-04 회원가입 폼 유효성 검사 구현      [FE] / 조하원
  │
  └── (버그) 01-B01: 로그인 토큰 만료 시 갱신 없이 오류가 반환되는 문제
          Highest
          └── 01-B01-01 토큰 만료 예외 처리 및 자동 갱신 로직 수정  [BE] / 진혜린
```

## 6. Description 작성 형식

---

### 6-1. 스토리 Description

```
설명
{사용자 입장에서 기능 필요 이유 1~3줄}

인수 조건
- 완료 기준 1
- 완료 기준 2
- 완료 기준 3
```

**예시**

```
설명
관리자가 이메일과 비밀번호로 로그인하여 관리자 기능에 접근할 수 있다.

인수 조건
- 올바른 이메일/비밀번호 입력 시 로그인 성공
- 잘못된 정보 입력 시 오류 메시지 표시
- 로그인 성공 시 관리자 대시보드로 이동
- JWT 토큰 발급 및 로컬 저장소 저장
```

---

### 6-2. 작업 Description

```
설명
{구체적으로 무엇을 구현하는지 1~2줄}

테스트 전략
{검증 방법}

인수 조건
- 완료 기준 1
- 완료 기준 2
```

**예시**

```
설명
POST /api/auth/login — 이메일, 비밀번호를 입력받아 관리자 인증 처리

테스트 전략
단위 테스트: 유효 입력 시 200 응답 및 토큰 반환 확인
예외 테스트: 잘못된 비밀번호 입력 시 401 응답 확인

인수 조건
- 정상 입력 시 200 OK 및 JWT 토큰 반환
- 잘못된 비밀번호 시 401 Unauthorized 반환
- 존재하지 않는 이메일 시 404 Not Found 반환
```

---

### 6-3. 버그 Description

```
발생 환경
{OS, 브라우저, 서버 환경 등}

재현 방법
1. Step 1
2. Step 2
3. Step 3

기대 결과
{정상 동작 결과}

실제 결과
{실제 발생 문제}
```

## 7. 상태 전환 규칙

```
해야 할 일 → 진행 중 → 완료
```

- 생성 시 기본 상태: **해야 할 일**
- 브랜치 생성 또는 작업 시작 시: **진행 중**
- GitLab push 또는 작업 완료 시: **완료**

---

## 8. 이슈 생성 금지 사항

- 에픽 없이 스토리/버그/작업 단독 생성 ❌
- 스토리/버그 없이 작업 단독 생성 ❌
- 작업을 Subtask로 생성 ❌
- 에픽/스토리에 레이블 설정 ❌
- 에픽/스토리에 담당자 설정 ❌
- 스토리에 Story Points 미설정 ❌
- 작업에 Story Points 설정 ❌
- 서술형이 아닌 스토리 제목 ❌

```
(X) 로그인 구현
(O) 관리자는 로그인을 할 수 있다
```
